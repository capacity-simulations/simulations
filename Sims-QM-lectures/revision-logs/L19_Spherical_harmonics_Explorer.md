EC 04/07/26: Fixed a phase-coloring bug in the complex spherical-harmonics mode. The `computeComplexYPhase` function computed `arg(Y_l^m)` as `m·φ + π·[P<0]`, which is correct for m ≥ 0 but omits the `(-1)^|m|` factor from the relation Y_l^{-|m|} = (-1)^|m| [Y_l^{|m|}]* when m < 0. For odd negative m (m = −1, −3, −5) this produced a hue shift of π, giving visibly wrong phase coloring on the polar surface, point cloud, and projection views. The fix adds `if (m < 0 && |m| % 2 === 1) phase += π` after the base phase calculation. All other physics (associated Legendre polynomials, normalization, real harmonics, complex magnitude, coordinate axes, nodal counts) was audited against the lecture notes (QM_intro.tex §3.3) and found correct.

EC 04/09/26: The coordinate-axis styling and projection legend were cleaned up for visual consistency. The rendered `x`, `y`, and `z` axes and their labels were changed to white, and the tooltip text was updated so it no longer described the `z` axis as green. In projection mode, the legend text was also corrected to match the actual rendering: the old wording incorrectly implied a radial `|Y|` encoding even though the view uses a fixed sphere, so the legend now states only the positive color encodings that are actually shown (`Color` for real harmonics; `Hue` and `Brightness` for complex harmonics).
