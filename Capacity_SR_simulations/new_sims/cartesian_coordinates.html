About the simulation window

This simulation shows how distance stays the same when you translate or rotate a Cartesian coordinate system over a fixed pattern of stars.

Stars are fixed in the background. Use the rotation slider (or auto-rotate) to turn the axes, and read each star’s coordinates in the panel as the frame changes.

Green chords connect the constellation and label each edge length d. Those values do not change as you move or rotate the axes—the hallmark of a Euclidean invariant. The ds² table highlights the first three edges.

Reset restores the origin and rotation only. New Stars picks a different constellation.

Axis numbers use y upward (panel y = −y′/PX). Invariance is under translating/rotating the frame, not resizing the window (relayout rescales the pattern).

Rigid transformation

Δx = x − x₀,  Δy = y − y₀

x′ = cosθ·Δx + sinθ·Δy
y′ = −sinθ·Δx + cosθ·Δy

R(θ) = [ cosθ  sinθ ]
        [ −sinθ  cosθ ]

Panel: x = x′/PX,  y = −y′/PX

Euclidean invariant

ds² = Δx² + Δy²,  d = √ds²

Rotations use an orthogonal matrix with RTR = I. This is the classical stepping stone toward Special Relativity, where the spacetime interval replaces ordinary distance.
