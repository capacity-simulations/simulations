About: Frame Clock and Event Coordinates

Source

Lecture
5 — Spacetime
Simulation
1 — Frame clock and event time (guided inquiry)
Concept
Coordinate time is assigned to an event by the whole reference frame at once — it does not depend on the event's distance from any clock and does not wait for light to propagate.
Serves
Define events and reference frames in spacetime; appreciate the physical meaning of simultaneity; establishes the shared visual language (frame badge, single central clock, frame-coloured grid) reused by all later frame sims.
Inquiry
Why one clock per frame? · What time is stamped for an event at x = 10 when t = 0? · Should the sim wait for light? · Do two frames agree on an event's time?
Misconception
That the recorded time for a distant event should include the light-travel delay to the clock (encoded as prediction choice B).

Frames & symbols

▾
Frame S — rest frame (v = 0)
Frame S′ — moves at v relative to S
t = coordinate time in S · t′ = coordinate time in S′
x, y = S space coords · x′, y′ = S′ space coords
v = speed of S′ rel. to S (units of c) · γ = Lorentz factor = 1.091
HUD notation: The canvas HUD shows t′(x′=0) — the S′ central-clock reading at its own origin. Each event tag shows t′ = γ(t − vx), the full Lorentz-transformed time at that event's position. These generally differ when x ≠ 0.
Invariant vs frame-dependent: the event (a point in spacetime) is the same for both frames; its coordinates (x,t) and (x′,t′) are frame-dependent. v and γ are properties of the pair (S, S′).
Units: c = 1, so 1 grid-square = 1 length-unit = 1 time-unit.

Core claim
The event's S time tag uses the full transform t′ = γ(t − vx), which generally differs from the S′ central-clock reading t/γ at the same S-moment — that difference is the relativity of simultaneity. In Newtonian physics every frame stamps the same t; here, because the event has x ≠ 0, S and S′ disagree on its time coordinate.
